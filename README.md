# AI 自动阅卷桌面端

## 当前状态警告

这个软件目前还不成熟，现阶段仅适合继续开发和内部测试，不适合直接作为稳定阅卷工具投入正式使用。

当前最明显的问题：

- 批改正确率偏低
- 对学生手写答案的 OCR 识别非常不精准
- 复杂书写、连笔、涂改、模糊扫描时误识别较多

因此这个项目当前更接近“可运行原型”，下一步仍然需要继续优化 OCR、提示词、评分策略和整体批改稳定性。

这是一个用 Python 重写的桌面端阅卷工具，面向网页阅卷场景使用。软件通过截图识别框获取学生作答区域，再调用 AI 完成 OCR 与评分，最后把分数自动填入网页中的打分框并点击提交。

当前版本重点实现了桌面端流程：
- 按钮添加识别框、打分框、提交框
- 三种框使用不同颜色区分，并支持拖拽和缩放
- 仅识别识别框内的内容，不读取打分框和提交框
- 支持主评、副评、仲裁的独立开关
- 支持多服务商、多模型、每个服务商独立保存 API Key
- 支持评分材料图片作为参考输入
- 支持空白答题卡检测、连续批改、提交前确认、历史导出、人工纠错

## 主要界面

软件包含以下页面：

- `批改`：启动一次批改、连续批改、填分、提交、纠错、空白卡范本采集
- `配置`：主评/副评/仲裁设置、评分控制、OCR 控制、题目内容、参考答案、评分标准、评分材料图片
- `服务商`：维护供应商的接口地址、API Key、模型列表、当前模型、推理强度
- `操作框`：查看和调整识别框、打分框、提交框
- `历史`：查看批改结果并导出
- `方案`：保存、载入、导入导出整套配置

## 多服务商与模型

软件当前按 OpenAI 兼容接口组织服务商。每个服务商单独保存这些信息：

- 名称
- API 端点
- API Key
- 当前模型
- 模型列表
- 推理强度

当前内置了几组示例：

- `5plus1官方`
- `火山引擎`
- `OpenAI兼容`
- `自定义服务商`

你可以在 `服务商` 页：

- 新增服务商
- 复制已有服务商
- 删除服务商
- 保存该服务商的 API Key
- 测试该服务商连接
- 给该服务商维护多个模型

模型列表用逗号分隔，例如：

```text
gpt-4o,gpt-4o-mini,o3
```

配置页里的主评、副评、仲裁都可以分别选择：

- 使用哪个服务商
- 使用该服务商下的哪个模型
- 是否启用这一段流程

规则如下：

- 主评关闭：本次不能开始批改
- 副评关闭：仅使用主评
- 仲裁关闭：当主评与副评分差超过阈值时，直接取平均，不触发第三次仲裁

## 阅卷基本流程

1. 打开网页阅卷系统
2. 在 `操作框` 页点击 `+识别框`、`+打分框`、`+提交框`
3. 点击 `打开拖拽调整`，把三个框拖到网页对应位置
4. 在 `配置` 页填写题目内容、参考答案、评分标准
5. 如有需要，添加评分材料图片
6. 在 `服务商` 页配置 API Key、模型和接口
7. 回到 `批改` 页点击 `开始批改` 或 `连续批改`

识别与提交逻辑：

- 识别框：只截取学生作答区域
- 打分框：自动点击并输入最终分数
- 提交框：自动点击提交

## 评分材料图片

`配置` 页支持添加本地图片，作为评分参考材料一起发送给 AI。

适合放进去的内容：

- 题目截图
- 标准答案截图
- 评分标准照片
- 教辅材料截图

这些图片不会替代识别框截图，而是作为额外参考输入。

## 评分控制

当前支持的评分控制包括：

- 满分
- 取整步长
- 取整方式：`round` / `floor` / `ceil`
- 勤勉加分开关
- 勤勉最高加分
- 分小题评分表

如果配置了分小题：

- 软件会按小题组织提示词
- AI 返回的小题得分会单独记录
- 总分按小题总和和取整规则处理

## OCR 与空白卡

OCR 相关设置：

- OCR 预处理强度
- 识别框内边距

空白卡相关设置：

- 启用空白答题卡检测
- 采集空白卡范本
- 清除空白卡范本

识别区只会截取识别框内部，并可通过内边距进一步收缩区域，减少把框线、按钮、邻近题目一起截进去的情况。

## 批改模式

支持三种模式：

- `normal`：普通模式
- `trial`：试改模式，等待教师确认
- `unattended`：无人值守模式，按倒计时自动提交

还支持这些流程控制：

- 提交前弹出确认窗口
- 普通模式倒计时
- 无人模式倒计时
- 提交后等待下一份的时间
- 限制批阅份数

## 提交确认与纠错

当启用提交前确认时，批改完成后会弹出确认窗口，显示：

- 识别框截图
- 最终得分
- 小题得分
- 识别答案
- 评分依据

确认窗口可执行：

- 提交并下一份
- 暂停倒计时
- 取消提交
- 标记为空白卡
- 纠正分数

人工纠错支持：

- 手动修改最终分数
- 填写纠错理由
- 可同步修改参考答案和评分标准

## 历史与导出

历史页面会保存最近的批改结果，可导出：

- JSON
- CSV
- HTML

历史记录包含的核心信息：

- 识别答案
- AI 原始分数
- 最终分数
- 评语
- 小题得分
- 双评结果
- 勤勉加分

## 配置方案

`方案` 页支持：

- 保存为当前方案
- 另存为新方案
- 删除选中方案
- 导出全部配置
- 导入配置

导出配置时会一起保存：

- 服务商列表
- 每个服务商的 API Key
- 当前模型和模型列表
- 主评/副评/仲裁设置
- 评分规则
- 操作框位置

## 数据文件

程序数据保存在当前目录下的 `data/`：

- `data/config.json`：当前配置
- `data/presets.json`：方案集合
- `data/history.json`：历史记录
- `data/blank_reference.json`：空白卡范本

## 当前限制

- 当前版本主要面向 Windows 桌面网页阅卷场景
- 自动填分和点击提交依赖屏幕坐标，所以网页布局变化时需要重新调整框位置
- 外部 AI 服务默认按 OpenAI 兼容接口调用，不兼容该协议的服务商需要自行提供兼容网关
