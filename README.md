# AI 自动阅卷桌面端

> ⚠️ 当前仍处于内测阶段，整体批改准确率偏低，对学生手写答案的 OCR 识别也不够精准，后续仍需持续优化。

本项目是基于 Python 重写的阅卷助手桌面应用，目标是在桌面端复刻并拓展参考软件的全部功能。应用通过截图识别答题区域，再调用多家 AI 服务商提供 OCR 与评分能力，最终把分数自动回填到浏览器或打分系统。

## 主要能力

- **答题卡 / 打分框 / 提交框三类区域**：通过按钮添加，分别使用不同颜色边框显示，支持拖拽移动和缩放尺寸，方便对齐试卷版式。
- **多种工作流模式**：内置 `direct`（截图直接送审）与 `ocr_first`（先 OCR 再评分）流程，可按题型和供应商能力选择。
- **OCR 精准范围**：只在“识别框”内部执行 OCR，避免误识别打分框或提交按钮中的内容。
- **AI 供应商与模型管理**：支持多服务商并存，每家可配置多个模型，切换模型时可独立选择 API Endpoint 与断点设置，多套 API Key 会持久保存。
- **主评 / 副评 / 仲裁**：三种角色的开关可独立控制，灵活应对校内不同批改流程。
- **批改校对对话框**：提供“得分校正”窗口，老师可调整 AI 得分、更新参考答案与评分标准，并写入评分说明。
- **批改自动化**：支持按照 Tab / Enter / Space 等模式在页面上切换分数输入框，实现多小题连续录入。
- **批前检查与调试**：新增“检查配置”与“调试批改”按钮，一键验证截图区域、API Key、模型参数等是否齐备。
- **历史记录导出**：默认导出 CSV，同时在已安装 `python-docx` 与 `openpyxl` 时，额外生成 Word（.docx）与 Excel（.xlsx）报告。
- **十套评分模板**：内置 10 组评分标准，可根据年级、题型快速切换并应用。

## 环境要求

- 操作系统：Windows 10 及以上
- Python：3.10+
- 依赖：详见 `requirements.txt`

```powershell
pip install -r requirements.txt
```

默认依赖包括：`Pillow`、`requests`、`pyautogui` 等；若需 Word / Excel 导出能力，请额外安装 `python-docx` 与 `openpyxl`。

## 启动方式

```powershell
python main.py
```

也可以直接双击 `启动软件.bat` 快速启动。

## 使用流程概览

### 三步启动批改

#### 第一步：配置环境

1. **添加操作框**：在主界面通过按钮依次添加"识别框""打分框""提交框"，并拖拽调整位置和大小，使其精确对齐试卷版式。
   - **识别框**（绿色）：框住答题卡/答卷的文字答题区域，宽高应不小于 50×50px
   - **打分框**（蓝色）：框住浏览器/系统中填分数的输入框，宽高应不小于 30×20px
   - **提交框**（橙色）：框住提交按钮，宽高应不小于 40×20px

2. **配置 AI 参数**：在"配置"面板选择：
   - 启用的评分角色（主评、副评、仲裁）
   - AI 服务商和具体模型
   - 工作流模式：`direct`（截图直接评分）或 `ocr_first`（先 OCR 再评分）
   - 确保每个启用的服务商都填写了 **API Endpoint** 和 **API Key**

3. **设定评分依据**：填写以下至少一项：
   - 题目内容
   - 参考答案
   - 评分标准
   - 评分材料图片

4. **设置批改份数**：在配置面板的"目标份数"字段填入要批改的试卷份数（0 表示无限循环）。

#### 第二步：调试验证

使用"调试批改"进行单份试卷的测试：
   - 验证 OCR 识别结果是否准确（查看输出窗口）
   - 验证 AI 评分逻辑是否符合预期
   - 调试模式**不会**自动填分和提交，只看结果

#### 第三步：连续批改

配置全部完成后，点击"开始批改"：
   - 系统会进行**全量检查**：验证所有框的位置和大小、API 参数、评分依据等
   - 若有问题，会列出详细的错误提示，需要先修正
   - 检查通过后，进行**预截图质量检查**，验证当前识别框能否正常工作
   - 最后弹窗确认目标份数和所有配置，点确认即开始自动批改

自动批改流程中：
   - 自动截取识别框内容
   - 调用 AI 进行 OCR 识别与评分
   - 自动在打分框填入 AI 计算的分数
   - 自动点击提交框，完成提交
   - 等待指定延时后继续下一份，无需人工干预

#### 第四步：监控和导出

- 批改过程中可随时点击"停止批改"中断流程
- 若设置了目标份数，到达后自动停止
- 在"历史记录"面板查看全部批改结果
- 支持导出为 CSV、Word（.docx）、Excel（.xlsx）等格式

### 配置参数说明

- **取卡延时**：截取识别框前的等待秒数，给浏览器/摄像头稳定时间
- **打分延时**：AI 评分完成到自动填分之间的等待秒数
- **批改间隔延时**：完成一份试卷到开始下一份之间的等待秒数
- **多打分框切换**：在有多个小题打分框时的光标移动策略（单个/Tab/Enter/Space）

## 配置文件说明

`data/config.json` 中保存了所有界面设置与服务商配置：

- `workflow.recognition_mode`：`direct` / `ocr_first`
- `workflow.ocr_provider_name`：当前 OCR 供应商
- `providers`：包含每家服务商的模型列表、Endpoint、是否启用等信息
- `grade_level`、`subject`、`question_type`：可选Metadata，用于不同题型的模板切换
- `score_switch_mode`：自动分数录入时的光标切换策略

##目录结构

- `main.py`：应用入口
- `app.py`：主界面与核心业务逻辑
- `models.py`：配置、服务商、工作流等数据模型定义
- `ai_client.py`：AI 服务调用封装（OCR / 评分等）
- `automation.py`：自动化填分逻辑
- `history.py`：历史记录存储与导出
- `correction_dialog.py`：得分校正对话框
- `submit_dialog.py`、`overlay.py` 等：辅助界面组件
